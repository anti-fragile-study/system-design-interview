# Ch13. 검색어 자동 완성 시스템

### 트라이 자료구조에 대해 설명해주세요.

* 트라이 자료구조를 사용하여 검색어 자동완성 기능을 구현한다고하면 시간복잡도는 어떻게 될까요?
* 그렇다면 해당 시간 복잡도를 개선할 수 있는 방법이 있나요?

<details>
<summary><h4>해설</h4></summary>

> 트라이 자료구조에 대해 설명해주세요.
* 문자열들을 간략하게 저장할 수 있는 트리 형태의 자료구조입니다. 각 노드는 문자를 저장하고, 자식 노드들은 해당 문자 뒤에 올 수 있는 문자를 더해서 저장합니다. 이를 통해 문자열을 저장하고 검색하는데 유용합니다.

> 트라이 자료구조를 사용하여 검색어 자동완성 기능을 구현한다고하면 시간복잡도는 어떻게 될까요?
* 접두어의 길이를 p, 노드의 자식 노드의 개수를 c라고 했을 때 O(p) + O(c) + O(clogc)가 됩니다.

> 그렇다면 해당 시간 복잡도를 개선할 수 있는 방법이 있나요?
* 접두어의 최대 길이를 제한하거나 각 노드에 인기 검색어 캐싱을 통해 시간복잡도를 개선할 수 있습니다.
</details>

### 사용자가 검색을 할 때 마다 트라이 자료구조를 매번 수정한다면 어떤 문제가 발생할 수 있을까요?

* 해당 문제를 어떻게 해결할 수 있을까요?

<details>
<summary><h4>해설</h4></summary>

> 사용자가 검색을 할 때 마다 트라이 자료구조를 매번 수정한다면 어떤 문제가 발생할 수 있을까요?
* 매번 트라이를 갱신하므로 질의 서비스가 느려질 수 있습니다.

> 해당 문제를 어떻게 해결할 수 있을까요?
* 인기 검색어는 자주 바뀌지 않으므로 트라이는 자주 갱신할 필요가 없습니다. 검색어 로그를 통해 주기적으로 트라이를 생성하면 할 수 있습니다.

</details>
